# Обслуживание (Maintenance)

> **Цель документа**: Инструкции по регулярному обслуживанию проекта, обновлению зависимостей и внешних утилит.

---

## 1. Обновление yt-dlp

`yt-dlp` — критически важный компонент проекта. Видеоплатформы (YouTube, RuTube, VK и др.) часто меняют алгоритмы, поэтому регулярное обновление `yt-dlp` необходимо для стабильной работы скачивания.

### 1.1 Механизм обновления в приложении

Приложение предоставляет интерфейс для проверки и выполнения обновления `yt-dlp` без перезагрузки сервера (если позволяют права доступа в ОС).

- **Проверка версии**: Сервер выполняет `yt-dlp --version` и сравнивает с данными из GitHub API или через `yt-dlp --update-check`.
- **Кнопка в интерфейсе**: В Telegram Mini App (раздел "Система" или "Настройки") отображается текущая версия и кнопка "Обновить", если доступна новая версия.
- **Логика обновления**: При нажатии сервер выполняет команду обновления (например, `yt-dlp -U`).

### 1.2 Рекомендации по частоте обновлений

- **Автоматически**: Рекомендуется проверять наличие обновлений при каждом запуске сервера и раз в 24 часа.
- **Вручную**: Если скачивание конкретного видео завершается ошибкой `Sign-in confirmed`, первым делом следует нажать кнопку обновления в интерфейсе.

### 1.3 Ограничения в Docker

В Docker-контейнере обновление через интерфейс может быть ограничено (read-only filesystem). В этом случае обновление происходит путем пересборки/перезапуска контейнера с новой версией. См. [DEPLOYMENT.md](./DEPLOYMENT.md).

## 2. Обновление БД

Используйте Flyway для миграций.
- Все изменения схемы должны быть в `server/infra/src/main/resources/db/migration/`.
- Никогда не меняйте существующие файлы миграций, создавайте новые.

---

## 3. Мониторинг логов

Следите за ошибками в логах со специфическими кодами `YT_DLP_ERROR`.
Частые ошибки этого типа — явный сигнал к необходимости обновления `yt-dlp`.
